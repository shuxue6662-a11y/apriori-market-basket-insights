# Apriori Market Basket Insights

基于 **Apriori 关联规则算法** 的零售购物篮分析项目。使用 UCI Online Retail 数据集（英国某在线零售商 2010–2011 年交易数据），完成从数据清洗、探索性分析到关联规则挖掘与业务策略设计的完整链路。

> **核心成果**：从 54 万条原始记录中挖掘出 **709 个频繁项集** 与 **295 条强关联规则**，最高提升度达 **37.55**（REGENCY 茶具系列），并输出可落地的捆绑销售与交叉推荐策略。

## 📊 数据集与实验设置

- **数据来源**：UCI Machine Learning Repository — Online Retail Dataset
- **原始记录**：541,909 条
- **清洗后记录**：530,104 条（保留率 97.8%）
- **时间跨度**：2010.12 – 2011.12
- **订单数**：19,960 笔 | **商品数**：4,026 种 | **客户数**：4,338 位

**Apriori 参数设置**：

| 参数 | 取值 | 含义 |
|------|------|------|
| 最小支持度 | 0.015 | 组合至少出现在 299 笔交易中 |
| 最小置信度 | 0.4 | 条件概率至少 40% |
| 最小提升度 | 3.0 | 关联强度至少为随机购买的 3 倍 |
| 最大项集长度 | 3 | 最多挖掘 3-项集 |

## 💡 核心发现与业务策略

1. **系列化购买最强**：同系列不同颜色/款式商品关联度极高（如 REGENCY 茶具提升度 > 35），顾客具有强烈的成套购买倾向
2. **场景化驱动关联**：派对用品、烘焙用具等同一使用场景下的互补商品存在显著关联
3. **三类落地策略**：
   - **捆绑销售**：针对提升度 > 30 的系列商品推出套装（如英式下午茶三件套），预估客单价提升 2–3 倍
   - **智能推荐**：在购物车/详情页推送高置信度关联商品，预计关联商品销量提升 30%–50%
   - **场景陈列**：按使用场景（下午茶专区、派对用品专区）组织商品，激发连带购买

- **Python**: Pandas（数据清洗）、NumPy、mlxtend（Apriori / 关联规则）、NetworkX（规则网络图）、Matplotlib / Seaborn（可视化）

## 🚀 快速使用

### 环境依赖
```bash
pip install pandas numpy matplotlib seaborn mlxtend networkx openpyxl
```

### 运行分析
1. 下载 [UCI Online Retail 数据集](https://archive.ics.uci.edu/ml/datasets/online+retail) 并放置为 `Online Retail.xlsx`
2. 直接运行主脚本：
```bash
python Apriori关联规则分析_代码.py
```
3. 脚本将自动输出：
   - 5 张高清可视化图表（EDA / 规则分布 / 网络图 / 共现矩阵 / 提升度解读）
   - 终端业务策略报告（捆绑销售 / 智能推荐 / 场景陈列方案）

### 查看结果
- 打开 `关联规则结果.xlsx` 浏览 295 条规则明细
- 打开 `频繁项集结果.xlsx` 查看 709 个频繁项集
- 阅读 `本科生课程论文张弘.pdf` 了解完整研究背景与方法论
